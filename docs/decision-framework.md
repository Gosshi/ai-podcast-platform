# Decision Framework Models

本番組は「15分で、今日の“時間単価と支出効率”を決めるニュース」を提供する。
DeepDive と QuickNews の判断は、以下のフレームで行う。

## Frame A: 体験型コンテンツフレーム
- 適用対象: ゲーム体験版、買い切りコンテンツ、イベント参加
- 指標: 予想プレイ時間 ÷ 価格（補助指標として 1時間単価 = 価格 ÷ 予想プレイ時間）
- 基準:
  - 1時間単価 500円以下: 検討
  - 1時間単価 800円超: 見送り

## Frame B: サブスク整理フレーム
- 適用対象: 動画配信、音楽配信、月額サービス
- 指標: 月額 ÷ 月間視聴時間
- 基準:
  - 600円以下: 妥当
  - 1,000円超: 整理候補

## Frame C: セール／衝動課金フレーム
- 適用対象: 期間限定セール、クーポン、割引施策
- 指標: 実行予定日が確定しているか
- 基準:
  - カレンダーに入る: 検討継続
  - カレンダーに入らない: 買わない

## Frame D: 広告ストレスフレーム
- 適用対象: 広告付きプラン、広告表示仕様の更新
- 指標: 広告時間 ÷ 月間視聴時間
- 基準:
  - 15%超: 非広告プラン検討

## DeepDive 出力ルール
DeepDive の「5. 今日の判断（個人視点）」は、次の3要素を必須とする。
1. 使用フレーム名（Frame A/B/C/D）
2. 数値計算または条件判定
3. 結論

例:
- 「Frame A（体験型コンテンツフレーム）を適用。予想10時間÷3,000円で1時間単価300円。基準500円以下なので検討可。」

## QuickNews 出力ルール
- 判断タグ【今使う】【今使わない】【監視】は維持する。
- 可能な項目では Frame A/B/C/D を併記し、判断根拠を1行で示す。
- タグだけで終わらせず、時間単価または支出効率への影響を明記する。

## 運用メモ
- 同一回の DeepDive で同じテンプレ判断（例: 30分固定）を繰り返さない。
- フレームはトピック性質に合わせて切り替える（例: Steam=Frame A, Disney+=Frame B, Netflix広告=Frame D）。
