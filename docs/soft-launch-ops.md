# Soft Launch Ops

更新日: 2026-03-23  
対象サービス: `判断のじかん by SignalMove`

## 目的

soft launch の 6 週間で、

- 公開回の質を安定させる
- `公開回 -> 無料登録 -> trial -> paid -> 継続` を毎週同じ形で見る
- 改善を「思いつき」ではなく運用ループで回す

ための運用メモです。

前提:

- 公開回は手動作成・手動公開
- X は手動運用
- `daily-generate` と X 自動投稿は明示有効化まで manual-only
- 広告費は使わない

## 今週のタスク表

### 毎週の固定タスク

| 曜日 | やること | 出力 |
| --- | --- | --- |
| 月 | 前週レビュー | KPI 更新、勝ち筋メモ、次回テーマ候補 |
| 火 | 公開回 1 本目を準備 | title / description / judgment card / X 投稿文 |
| 水 | 公開回 1 本目を公開 | Apple / Spotify / Web / X 反映確認 |
| 木 | 24 時間レビュー | 初動の反応確認、次回 copy 修正 |
| 金 | 公開回 2 本目を準備・公開 | 同上 |
| 土 | X まとめ投稿 | 今週の公開回まとめ |
| 日 | 翌週テーマ仮決め | 次週の公開候補 2-3 本 |

補足:

- 最初の 2 週間は `週 2 本` が上限でよい
- 質が不安な週は `週 1 本` に落としてもよい
- `毎朝` を約束するより、守れる頻度を優先する

## 公開前チェック

### 公開回

- title が 1 つの迷いに絞られている
- description 1 文目で判断対象が伝わる
- judgment card 見出しが具体語になっている
- `AI最新動向` `話題まとめ` のような抽象見出しを避ける
- 公開 permalink が正しく開く
- 音声が再生できる

### 配信面

- feed item title が期待どおり
- feed item description が generic でない
- Apple / Spotify で見えて困る文言がない
- X 投稿 1 行目が title と噛み合っている

### CTA

- 公開回ページに無料登録の理由がある
- 今回の回で何が判断できるかが見える
- 有料導線が強すぎず弱すぎず置かれている

## KPI ダッシュボード項目

### 毎週記録する数字

| 指標 | 意味 | 目安 | メモ |
| --- | --- | --- | --- |
| 公開回数 | 今週何本出したか | 1-2 本 | 無理に増やさない |
| 公開回再生数 | その回がどれだけ聴かれたか | まず観測 | Apple / Spotify / Web 合算でざっくりでもよい |
| 公開回 -> 無料登録率 | 公開回が登録につながったか | まず 3-5% を観測 | 導線の強さを見る |
| 無料登録 -> trial 開始率 | 無料体験に進んだか | まず 10% 以上を観測 | 有料 CTA の強さを見る |
| trial -> paid | 7日後に有料化したか | 初期 10-18% を観測 | 価格より価値訴求を見る |
| 4週継続率 | 有料が続いたか | まず 50% 以上を観測 | 継続価値の確認 |

### 週次レビューで必ず書くこと

- 今週一番反応が良かった回
- その title / description / judgment card の何が効いたか
- 反応が弱かった回
- 何を次回で変えるか
- 次週の仮説 1 つ

## 6 週間カレンダー

### Week 1

- 公開回を 1-2 本出す
- 計測の型を固定する
- title / description / judgment card の人間レビューを必須化する

### Week 2

- 公開回を 3 本以上にする
- X 手動運用を固定する
- どのテーマに初動があるかを見る

### Week 3

- title の勝ち筋を整理する
- 公開回ページの CTA が弱くないかを見る
- `公開回 -> 無料登録` が鈍ければ copy を直す

### Week 4

- `trial -> paid` の初回実測を見る
- 有料訴求の弱さがあれば onboarding と説明文を見直す

### Week 5

- 反応の良いテーマに寄せる
- 弱いテーマは一度止める
- 公開回の長さと焦点を見直す

### Week 6

- 続行 / 要修正 / 見直し を判定する
- 続行なら `public podcast / member-only episode` 分離設計を次フェーズに上げる
- 未達なら導線と有料価値を先に修正する

## 運用中にできる改善

### 1. title 改善

症状:

- 公開回はあるのに再生が弱い

やること:

- 先頭に判断対象を置く
- 抽象語を減らす
- `悩み名 + 判断` に寄せる

例:

- `AI最新動向`
  - 弱い
- `そのAIツール、今月も課金する？`
  - 良い

### 2. description 改善

症状:

- 再生はされるが登録に進まない

やること:

- 1 文目で何を判断する回かを書く
- 2 文目で比較軸を書く
- 3 文目で何が持ち帰れるかを書く

### 3. judgment card 見出し改善

症状:

- 公開回ページで中身が薄く見える

やること:

- `最新動向` `注目ポイント` を避ける
- `何を残すか` `どこでやめるか` `何を比べるか` に寄せる
- 見出し単体で判断対象が分かるようにする

### 4. CTA 改善

症状:

- 登録はあるが trial に進まない

やること:

- 有料で何が深くなるかを明文化する
- `公開回の続き` ではなく `判断を実行に落とす` 価値を出す
- Replay / Alerts / 履歴 のどれが効くかを観察する

### 5. 継続価値改善

症状:

- trial は始まるが paid や継続が弱い

やること:

- 公開回と会員価値を分けて考える
- 会員向けは行動提案と見直しタイミングを強くする
- `public podcast / member-only episode` 分離を次フェーズ候補に置く

## 実験ルール

- 1 週間で大きい変更は 1 つまで
- 複数の要素を同時に変えすぎない
- 仮説を書いてから変える
- 勝ち筋が出るまでは価格をいじらない
- 広告や自動化は、trial 開始率の傾向が見えるまで入れない

## 続行判断

### 続行

- 公開回から trial が動く
- paid が少数でも出る
- 継続にゼロではない兆しがある

### 要修正

- 再生はあるが登録が弱い
- trial はあるが paid が弱い
- 回ごとの quality variance が大きい

### 見直し

- 再生自体が伸びない
- trial がほとんど動かない
- 手動運用の負荷が高すぎて頻度を守れない
