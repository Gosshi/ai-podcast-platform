# Public Episode Copy Template

更新日: 2026-03-23  
対象サービス: `判断のじかん by SignalMove`

## 目的

Apple Podcasts / Spotify / Web / X で同じ公開回が見られる前提で、
毎回の `title` `description` `judgment card 見出し` `X 投稿文` を
手動公開でもブレなく作るための運用テンプレートです。

狙いは 3 つです。

- 一目で「何を判断できる回か」が分かる
- Apple / Spotify の一覧でも generic に見えない
- Web の公開回から無料登録・有料開始に自然につながる

## タイトルの基本ルール

- `1回につき1つの迷い` に絞る
- 先頭に `判断対象` を置く
- 後半に `どう決めるか` を置く
- `Daily Topic` `JA` `日付` のような管理用語を出さない
- 目安は `18〜32文字`

## タイトルの型

### 型 A: 見直す型

`[判断対象]を見直す`

例:

- `サブスクとゲームの使い方を見直す`
- `AIツール課金の続け方を見直す`
- `読む記事を絞る基準を見直す`

### 型 B: 今やるか型

`[判断対象]、今やる価値ある？`

例:

- `新しいAIツール、今やる価値ある？`
- `見逃した映画、今週観る価値ある？`
- `その有料プラン、今月入る価値ある？`

### 型 C: 使うべきか型

`[判断対象]に時間 / お金を使うべきか`

例:

- `そのサブスクに時間を使うべきか`
- `話題のゲームにお金を使うべきか`
- `その記事に読む時間を使うべきか`

### 型 D: どれを残すか型

`[判断対象]、残すならどれ？`

例:

- `動画サブスク、残すならどれ？`
- `AIツール、残すならどれ？`
- `読みたいニュース、追うならどれ？`

## title を決める手順

1. その回で一番強い判断対象を 1 つ選ぶ
2. リスナーが日常で使う言葉に直す
3. 上の型 A-D のどれかに当てる
4. 読んだ瞬間に判断対象が分かるか確認する

## description の基本ルール

- 2〜3文で終える
- 1文目で `何を判断する回か`
- 2文目で `何を比べるか`
- 3文目で `聴いたあと何が持ち帰れるか`
- 具体語を入れる
  - 時間
  - 料金
  - 利用頻度
  - 比較軸
- `最新情報をお届けします` のような中身の薄い言い回しは避ける

## description の型

`[判断対象]について、[比較軸1] と [比較軸2] から、今やるか見送るかを整理します。`

`今回は [時間 / 料金 / 利用頻度 / 代替手段] を見ながら、残す・止める・保留の基準を短く深く確認します。`

`聴いたあとに、今日の次の一手を決めやすくするための公開回です。`

## description 例

### サブスク系

`動画サブスクとゲーム課金について、利用頻度と時間単価から、今月も続けるか見直すかを整理します。今回は残す・止める・保留の基準を短く確認し、通勤中の7分で次の一手まで決めやすくします。`

### AIツール系

`新しいAIツール課金について、料金と時短効果、乗り換えコストから、今試すか見送るかを整理します。勢いで課金せずに判断するための比較軸を、公開回で短く確認できます。`

### 情報収集系

`話題の記事やニュースに読む時間を使うべきかを、緊急度と再現性、代替手段から整理します。今読む・後回し・スルーの基準を短く確認し、情報疲れを減らすための回です。`

## judgment card 見出しの基本ルール

- `1カードにつき1判断` に絞る
- 見出しだけで `何を残すか / 何を止めるか / 何を保留するか` が分かる
- `use_now / watch / skip` の badge と合わせて読んだ時に意味が増える
- `最新動向` `注目ポイント` `比較結果` のような抽象語を避ける
- 目安は `14〜28文字`

## judgment card 見出しの型

### 型 A: 残す型

`[判断対象]だけ残す`

例:

- `毎週使う主力ツールだけ残す`
- `今月観る配信サービスだけ残す`
- `読み返す価値がある記事だけ残す`

### 型 B: 止める型

`[判断対象]は今やめる`

例:

- `惰性で払うツールは今やめる`
- `見ていないサブスクは今やめる`
- `追う理由が薄い作品は見送る`

### 型 C: 保留型

`[判断対象]は1つだけ保留する`

例:

- `比較中の新規ツールは1つだけ保留する`
- `次に試す配信先は1つだけ保留する`
- `今週やる候補作は1本だけ保留する`

## X 投稿文の基本ルール

- 1 行目は `title` をそのまま使う
- 2 行目で `何を比べる回か` を入れる
- 3 行目で `聴いたあと何が決まるか` を入れる
- 最後に `公開 URL` を置く
- ハッシュタグは `0〜1個` までに抑える

## X 投稿文の型

`[title]`

`[比較軸1] と [比較軸2] から、今やるか見送るかを整理する回です。`

`聴いたあとに、[残す / 止める / 保留] の線引きを決めやすくします。`

`[公開 URL]`

## 避ける表現

- `Daily Topic 2026-03-22`
- `Japanese episode for ...`
- `最新情報をテンポよくお届けします`
- `AIが自動生成したポッドキャストです`
- 論点を 3 つ以上並べて焦点がぼける説明
- `注目ポイント`
- `比較結果`
- `気になる話題を整理`

## 人間レビューで落とす条件

- title を他テーマに差し替えても成立する
- description に `料金 / 時間 / 利用頻度 / 代替手段` の具体軸がない
- judgment card 見出しが抽象名詞だけで終わっている
- X 投稿の 1 行目が title より弱い
- X 投稿を読んでも `何が判断できる回か` が分からない

## 公開前チェック

- title だけで判断対象が伝わる
- description の 1 文目に判断対象がある
- judgment card 見出しだけで判断対象が分かる
- 日付や言語コードが表に出ていない
- Apple / Spotify 一覧で generic に見えない
- X 投稿の1行目にもそのまま使える

## 推奨運用

- 公開前に `title / description / judgment card 見出し / X 投稿文` を同時に決める
- タイトルが弱い時は、まず `何を判断する回か` を 1 つに絞る
- description が弱い時は、`時間 / 料金 / 利用頻度 / 代替手段` のどれを比較したかを足す
- judgment card 見出しが弱い時は、`残す / 止める / 保留` の動詞を見出しに戻す
- X 投稿文が弱い時は、2 行目に `比較軸`、3 行目に `聴いたあと何が決まるか` を足す
