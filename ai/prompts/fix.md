# Codex Fix Prompt

レビュー結果をもとに、このリポジトリへ最小修正を実装してください。

## 使い方

1. `README.md` と `ai/README.md` を読む
2. 対象 run log を読む: `ai/runs/YYYY-MM-DD-<topic>.md`
3. `レビュー結果` と `修正対象` を確認する
4. 出力は `ai/templates/fix-plan.md` に沿って整理する
5. 実装後、run log の `修正内容` に反映する

## 実装方針

- 問題を整理し、`P0 / P1 / P2` で優先順位を付ける
- まずは最小修正でユーザーの迷いを減らす
- 既存 UX を壊さない
- 既存の状態モデル、ルーティング、free / paid gating を不用意に広げない
- 変更範囲は今回の問題解決に必要な場所へ限定する

## このプロジェクトで特に守ること

- `Decision Assistant` としての判断支援価値を強める
- `Judgment Card` の意味と CTA を明確にする
- 保存 / 履歴 / 結果 / alerts の流れを壊さない
- free / paid の差は成果ベースで見せる
- mobile での可読性と操作性を落とさない

## 実装時の作業ルール

- 変更前に対象問題を短く整理する
- 影響範囲を明示する
- build / lint / test を可能な範囲で実行する
- 失敗した検証は隠さず記録する
- run log に修正内容と確認結果を残す

## PR ルール

- PR タイトルは変更内容が一目で分かるものにする
- PR 本文には背景、何を直したか、確認内容、残課題を書く
- レビュー指摘と実装内容の対応が追えるようにする

## 出力

- `ai/templates/fix-plan.md` に沿った修正計画
- 実装内容
- 実行した確認コマンドと結果
- 残るリスク
