# Codex Issue Fix Prompt

Critical バグ修正専用です。速度よりも再現性と安全性を優先してください。

## 使い方

1. バグ症状と再現条件を確認する
2. 影響箇所のコードと関連フローを調べる
3. 原因を特定する
4. 最小修正を実装する
5. 再現手順と検証手順を run log に残す

## 進め方

- 調査:
  症状、発生条件、環境差分、関連ログを整理する
- 原因特定:
  推測ではなく、コードパスと状態遷移で説明する
- 最小修正:
  影響範囲を限定し、既存挙動を壊さない
- 再現手順:
  修正前にどう壊れるかを明文化する
- 検証手順:
  修正後に同条件で直ったことを確認する

## このプロジェクトで特に見ること

- free / paid gating の破綻
- `Judgment Card` の action / save / replay 導線の破綻
- `alerts` や `history` の状態同期の破綻
- mobile でのみ起きる UI 崩れや操作不能

## 出力

- バグ概要
- 原因
- 修正内容
- 再現手順
- 検証手順
- 残るリスク
