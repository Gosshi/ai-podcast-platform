# Codex Review Prompt

このプロジェクト `ai-podcast-platform` を、`Decision Assistant` として成立しているかを基準にレビューしてください。

## 使い方

1. `README.md` と `ai/README.md` を読む
2. 対象 run log を読む: `ai/runs/YYYY-MM-DD-<topic>.md`
3. 必要なら対象画面と関連コードを確認する
4. 出力は `ai/templates/review-report.md` に沿ってまとめる
5. 結果を run log の `レビュー結果` に追記する

## レビュー対象

- 初見理解:
  何のサービスか、誰のためか、何をすべきかが最初の 30 秒で伝わるか
- 初回導線:
  `/` → `/onboarding` → `/decisions` の流れが自然か
- ホーム画面:
  入口として十分か。価値、CTA、次の行動が明確か
- Judgment Card:
  意味、価値、アクション、free / paid 差分が一目で分かるか
- ナビゲーション:
  `Home / Decisions / Library / History / Account` の役割が明確か
- モバイル UX:
  狭い画面でも読めるか、操作しやすいか、情報が詰まりすぎていないか
- 課金 UX:
  free / paid の差が機能一覧ではなく成果ベースで伝わるか
- 継続利用 UX:
  保存 / 履歴 / 結果 / alerts が再訪理由としてつながっているか

## このプロジェクト固有の観点

- `Decision Assistant` として「今日どうするか」の判断支援になっているか
- `Judgment Card` の意味が UI 上の文言だけで伝わるか
- 保存 / 履歴 / 結果 の状態モデルが自然で、初見でも追えるか
- `alerts` が毎日開く理由になっているか
- free / paid の差が「見える成果」で理解できるか
- mobile でも主要タスクがストレスなく完了できるか

## 必須出力

- UX スコア
- 問題トップ 10
- 今すぐ直すべき UX 5 件
- 重大な仕様矛盾や状態遷移の破綻
- 既存導線を壊さず直せる最小修正の方向性

## 評価ルール

- 表面的な文言指摘だけで終わらせず、判断導線・状態遷移・継続利用の観点まで見る
- 問題ごとに「どこで」「誰が」「何に迷うか」を明示する
- 優先度は `P0 / P1 / P2` で付ける
- 不明点は推測で埋めず、前提不足として明記する
