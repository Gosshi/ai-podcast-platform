# Codex Verify Prompt

修正後の確認を行ってください。今回の変更点だけでなく、全体 UX と回帰も再確認してください。

## 使い方

1. `README.md` と `ai/README.md` を読む
2. 対象 run log を読む: `ai/runs/YYYY-MM-DD-<topic>.md`
3. `レビュー結果` `修正対象` `修正内容` を確認する
4. 変更画面と関連フローを再確認する
5. 出力は `ai/templates/verification-report.md` に沿ってまとめる
6. 結果を run log の `検証結果` と `残課題` に反映する

## 検証観点

- 今回の修正対象が本当に改善されたか
- 変更した場所以外に回帰がないか
- `Decision Assistant` としての理解しやすさが上がったか
- `Judgment Card` の意味と行動導線がより自然になったか
- 保存 / 履歴 / 結果 / alerts の状態遷移に不自然さがないか
- free / paid の差が成果ベースで伝わるか
- mobile で破綻していないか

## 必須出力

- 修正結果
- 回帰有無
- まだ残る問題
- 次に着手すべき項目

## 評価ルール

- 「直ったように見える」ではなく、利用シナリオで再確認する
- 変更差分だけでなく、関連画面も見る
- 問題が残る場合は severity と再現条件を明記する
